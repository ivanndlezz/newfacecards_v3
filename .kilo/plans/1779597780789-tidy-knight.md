# Plan: Immersive WYSIWYG Card Appearance Editor (Card Carousel 3 / Premium v2) — North Star: card-carousel-2.html

**Status**: New planning session (2026-05-23). Replaces/ supersedes the previous "silent-star" implementation which the user described as "horrible" because the previews were too minimal and not immersive/WYSIWYG enough.

**User explicit decisions captured in this session** (via structured clarification):
- Presentation: **A — Full overlay / slide-up editor** for maximum immersion (large tall carousel cards ~380-480 px high, matching the feeling and proportions of card-carousel-2.html as closely as possible).
- Personalization scope for v1: **Core only** (live font family + accent color tweaks on the active carousel preview, with real-time reflection in the live #card where possible). Custom image upload and text content overrides deferred.

## Goal

Deliver a **production-grade, stunningly immersive "Appearance Editor"** inside the admin that feels as premium, fluid, and WYSIWYG as the inspirational `card-carousel-2.html` lockscreen customizer — but fully adapted to the 6 real NFC card themes (`aurora`, `zen`, `tropico`, `obsidiana`, `brutal`, `deco`).

When the user activates **"Apariencia"** (or "Perfil") via the island menu while in edit mode:
- A beautiful full (or near-full) slide-up / overlay editor appears, closely mirroring the structure, glassmorphism, 3D perspective carousel, state machine ("Elegido" represented by the persistent live #card + "Seleccionando" represented by the large editor), bottom controls, and Personalizar panel from -2.html.
- Each carousel item is a **large, tall, high-fidelity live preview** of its corresponding real card theme — instantly recognizable, using rich visual signatures (mesh gradients, Japanese minimal red accent bar + square avatar, tropical sunset layers, volcanic glass, brutal concrete, art-déco lines, etc.).
- Font + accent color changes made in the Personalizar panel **instantly mutate the active carousel card's preview** (WYSIWYG) and, where safe, the live #card above.
- Selection ("Listo" / tap a card) applies the base theme + current overrides to the real card via the existing binder bridge, with persistence.
- The experience is "wow" iOS-polished, touch-friendly, keyboard accessible, reduced-motion safe, and 100% vanilla / self-contained (no Tailwind in production code).
- The live `#card` (binder target) is **never destroyed** — the entire editor lives in its own isolated DOM subtree (overlay or appended container).

This replaces the current small/thumbnail-based advanced carousel (card-carousel-3d) with the ambitious immersive version the user actually wants.

## Proposed Architecture

### Core Principle
**"Immersive Appearance Editor" as a new lazy-loaded, self-contained premium module** (inspired directly by -2.html structure) that is activated from the existing island "apariencia" flow.

- The small thumbnail carousel (current v2 or old 3d) can remain for simple cases or be deprecated in favor of the immersive editor.
- New module: `shared/components/appearance-editor/` (or `card-carousel-premium/` with editor sub-layer) containing:
  - `appearance-editor.js` (main orchestrator: state machine, overlay creation, event wiring to island/binder)
  - `appearance-editor.css` (full port of glassmorphism, 3D perspective, bottom sheet, personalization panel styles — pure vanilla)
  - `card-preview-renderer.js` (or internal) — the high-fidelity WYSIWYG preview engine for the 6 themes
  - Internal 3D scroller logic (reuse/adapt the best parts of current 3d.js + direct ports from -2.html's `renderCarousel`, `update3DEffects`, `setupIntersectionObserver`)
- Still behind dynamic `import()` for perf.
- Self-injects its CSS + reuses Tabler icons already present in admin (via island).
- No global pollution.

### Preview Fidelity Strategy (Critical for Immersion & WYSIWYG)
Recommended: **Hybrid "Live Preview Renderer" + scoped hero clones**

1. For each theme, maintain (or generate) a **"preview hero fragment"** — a self-contained mini-HTML + critical CSS that faithfully reproduces the most recognizable 60-70% of the card (hero/avatar/name/title/social or key decorative elements) at the scale needed for a tall carousel card.
2. At runtime the renderer injects the fragment into each large carousel item inside a scaled container (`transform: scale(0.55)` or tuned per theme) with `pointer-events:none`.
3. For **live core personalization** (font + accent):
   - The renderer exposes `updatePreview(themeId, { font, accentColor })`.
   - It either rewrites inline styles on the cloned elements or injects a tiny scoped `<style>` override block inside the preview subtree.
   - Accent color is mapped to the theme's primary variables (e.g. `--aurora-accent`, border colors, gradient stops, text highlights).
4. To keep in sync with real templates long-term: the preview fragments live alongside the full templates (new `templates/cards/previews/aurora.html` etc.) or are extracted at build time (but since no build: we hand-author/maintain the hero slices initially, with clear "update checklist" in docs).
5. Optional future path (documented): full runtime fetch + aggressive selector rewriting + shadow DOM for zero maintenance drift.

This delivers the "instantly looks like my real card" + live mutation feeling of -2.html without violating isolation.

**Large card dimensions for immersion**: ~240-280 px wide × 380-460 px tall (tuned), with generous horizontal padding in the scroller so only 1-2 neighbors are partially visible — exactly like the prototype.

### State Machine (Direct Port from -2)
- **Elegido / Applied state**: represented by the always-visible live `#card` (binder) at the top of admin. It shows the currently committed theme + overrides.
- **Seleccionando state**: the full slide-up editor (carousel + controls) appears when island mode === 'apariencia'.
- **Personalizando sub-state**: the bottom sheet (font + color grid + custom picker) slides up over the carousel when "Personalizar" is tapped on the active card. Changes mutate the active preview + (optionally) live-patch the main #card.
- "Cancelar" / Escape → close editor, discard transient overrides.
- "Listo" / tap card → commit, call enhanced `__applyCardTheme(themeId, {font?, accentColor?})`, hide editor, update persistence.

### Live WYSIWYG Bridge to Binder (Core Only Scope)
- Extend `window.__setCardTheme` / add `window.__applyCardTheme(themeId, overrides = {})`.
- The binder (or a thin wrapper) loads the base template, then after hydration injects a dynamic `<style id="live-theme-overrides">` that targets the root card classes with the chosen font/accent (using CSS variables or direct rules).
- For the duration of the editor session we can also do lightweight direct DOM patches on the live #card (safe because we control the session) so the user sees **exact final result** while tweaking — this is the ultimate WYSIWYG the user loves in -2.
- On final apply we persist `APP_THEME_SELECTED` + a new `APP_THEME_OVERRIDES` (JSON) in localStorage and (future) sync to card-data.

### Overlay / Slide-up Implementation
- On first "apariencia" activation in edit mode: dynamically create an absolute/fixed full-bleed or 90% height editor container (z-index above island but below toasts if needed).
- It contains the exact chrome from -2 (top nav bar with Cancel / title / Listo, the perspective carousel scroller, page dots, bottom action bar with Personalizar + Añadir, the slide-up personalizePanel).
- The live #card area can be visually "frozen" or slightly dimmed behind a backdrop for focus.
- On close: remove the editor DOM, restore any transient styles.
- This gives the exact "entering the design tool" feeling.

### Tech & Consistency
- Pure vanilla JS + CSS, self-contained injection (check existing link).
- Glassmorphism tokens extended from island-menu + direct ports of -2's `.glassmorphism`, `.glassmorphism-dark`, smooth cubic-bezier transitions, perspective 1200px, no aggressive scale on focus (per prototype wisdom).
- Use Tabler icons (`ti ti-*`) already loaded by island.
- Reduced-motion: completely flat 2D scroller + disabled perspective/rotate + simplified sheet animations.
- ARIA, keyboard (arrows, escape, enter), touch snap, raf-throttled live 3D updates — all ported.
- Still emits (and listens for) the `card-theme-selected` contract for compatibility; adds richer detail for overrides.

## Risks & Mitigations

- **Vertical space & admin layout disruption** (full overlay on a page that already has a tall #card): Mitigate with 85-95% height sheet + subtle backdrop, clear "applied to the card above" visual language.
- **Maintenance of 6 preview fragments** vs real templates drifting: Clear docs + checklist; long-term runtime extraction path.
- **Performance of rich large previews + live mutations + 3D**: Strict lazy load; requestAnimationFrame; only the active + immediate neighbors fully rich at start.
- **Style isolation during live tweaks**: All mutations happen inside the preview subtree or via carefully scoped CSS vars / injected style tags.
- **Cross-browser 3D + backdrop-filter**: Test matrix; graceful degrade on old Safari.

- Opening "Apariencia" in the admin produces an immersive slide-up editor whose visual and interaction quality matches or exceeds the user's experience with card-carousel-2.html.

- Standalone demo `card-carousel-3.html` delivers the same "wow" in isolation.
